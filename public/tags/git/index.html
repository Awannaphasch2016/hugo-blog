Git Workflow For Team

I intend to make this blog be a work-in-progress where I will keep adding useful git workflow when working with a team. Pull Requests To do a pull-requests, you need to do the following fork original repo. create new local branch and checkout to the new branch. commit new changes. send pull requests from new local branch from forked repo (origin) to original repo (upstream). Alternative to the pull-request workflow I mention above, it is possible to do pull-requests by using git forge 1, adding commit to new branch 2, and then do a pull requests to the target branch in remote.
