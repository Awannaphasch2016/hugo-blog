By Anak Wannaphaschaiyong

  • Edit log

I was following “What’s A Docker Image Anyway?" 1, but found that its content is no longer true, there are new changes to metadata of docker image, so I write this blog to reflect the changes that I observed at the time of writing.

docker -v
Docker version 20.10.17 build 100c701

I write the following Dockerfile.

FROM ubuntu

RUN echo hihi
RUN touch /hi
RUN rm /hi
CMD ["echo", "hello"]

I build container from Dockerfile above as followed

docker build -t scratch/example1

This can be validated by running the following.

docker images

Docker image contains information on what changed to the images it’s based on. Each image has a parent unless the image is “scratch” image 4.

  • container images compose of base OS + application + user libraries.
  • base OS is not a full-blown OS.
  • images contains binaries and data in a read-only files system.
    • a read-write layer is added when the contianer runs.
